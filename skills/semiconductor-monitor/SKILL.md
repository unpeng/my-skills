# 588170 操盘指导手册

## ⚠️ 强制约束：动态计算规则

**本技能中所有数值必须动态获取或计算，禁止硬编码。**

### 变量定义（运行时从用户和数据源获取）

```
[持仓数量] = 用户确认的持仓份额（首次必须询问）
[加权成本] = 用户确认的加权平均成本价（首次必须询问）
[可用资金] = 用户确认的可用现金（首次必须询问）
[最大亏损] = 用户确认的最大可承受亏损金额或比例（首次必须询问）
[昨收价]   = 从 qstock 获取的前一交易日收盘价
[当前价]   = 从 qstock 获取的实时/最新价格
[60日最高] = 从 qstock 历史K线计算的60日最高价
[60日最低] = 从 qstock 历史K线计算的60日最低价
[20日均量] = 从 qstock 历史K线计算的20日平均成交量
[竞价价]   = 9:15-9:25集合竞价阶段从 qstock 获取的实时竞价成交价
             （竞价阶段结束后即为开盘价）
[开盘价]   = 9:30开盘后从 qstock 获取的当日第一笔成交价（即K线的"今开"）；
             集合竞价结束时的[竞价价]就是[开盘价]，两者是同一个值在不同
             时间点的称呼，不需要重复获取
```

### 动态计算公式（禁止替换为固定值）

```
[回本价]     = [加权成本]
[止损位]     = [加权成本] × (1 - [最大亏损比例])  或  [加权成本] - [最大亏损金额]÷[持仓数量]
[做T买入位]  = [当前价] × 0.98（实时计算）
[做T卖出位]  = [当前价] × 1.02（实时计算）
[昨收+2%]   = [昨收价] × 1.02
[昨收+3%]   = [昨收价] × 1.03（比[昨收+2%]更强的确认阈值，用于10:00方向
             确认时判断涨势是否延续，见"情景B/C"）
[昨收-2%]   = [昨收价] × 0.98
[昨收-4%]   = [昨收价] × 0.96
[成本+2%]   = [加权成本] × 1.02
[成本-2%]   = [加权成本] × 0.98
[成本-4%]   = [加权成本] × 0.96
[持仓市值]   = [当前价] × [持仓数量]
[浮动盈亏]   = ([当前价] - [加权成本]) × [持仓数量]
[盈亏比例]   = ([当前价] / [加权成本] - 1) × 100%
[距回本]     = ([加权成本] / [当前价] - 1) × 100%
[止损亏损]   = ([加权成本] - [止损位]) × [持仓数量]
[资金上限比例] = 80%（默认值，做T单次最多使用的可用资金比例，见"做T操作
             方案"，对应 `monitor` 命令内部的 t_cash_cap_pct 参数）
[做T可用资金上限] = [可用资金] × [资金上限比例]
[做T可买份数] = [做T可用资金上限] ÷ [做T买入位]（向下取整，标准做T场景，
             见"做T操作方案"）
[挂单可买份数] = [做T可用资金上限] ÷ 实际挂单价（向下取整，用于非标准做T
             买入位的挂单场景，如按[昨收-2%]等价位挂单买入时，见"第四步
             情景D/E/F"）
```

### 执行规则

1. **每次使用技能必须先获取变量值**，不能跳过
2. **所有价格判断必须用变量**，不能写"1.31"、"1.27"等固定数
3. **输出给用户时显示变量的实际值**，如"当前价1.292，做T买入位1.266"
4. **做T份额根据可用资金计算**：[做T可买份数] = [做T可用资金上限] ÷ [做T买入位]

5. **每次输出结论都必须附带风险提示**："以上仅供参考，不构成投资建议，市场

6. **不构成投资建议**：本技能所有输出均基于历史数据和技术指标的规则计算，

### 第三步：集合竞价决策（9:15-9:25）

集合竞价期间，运行 qstock 获取竞价价格，计算涨跌幅。

**决策树（变量版）：**

```
集合竞价价格 = [竞价价]
涨跌幅 = ([竞价价] - [昨收价]) / [昨收价] × 100%

│
├─ 情景1：[竞价价] > [成本+2%]
│   │  判断：超预期高开，可能冲高回落
│   │  操作：不追高，等回调
│   └─ 开盘后回落到[加权成本]以下再考虑
│
├─ 情景2：[加权成本] < [竞价价] ≤ [成本+2%]
│   │  判断：接近回本，获利盘可能出逃
│   │  操作：观望，不操作
│   └─ 如果放量站稳[加权成本]，可持有
│
├─ 情景3：[昨收+2%] < [竞价价] ≤ [加权成本]
│   │  判断：反弹延续，但距回本仍有距离

│   └─ 涨到[加权成本]附近考虑减仓
│
├─ 情景4：[昨收-2%] < [竞价价] ≤ [昨收+2%]
│   │  判断：中性，等待方向选择
│   │  操作：不动，等方向
│   └─ 跌到[昨收-4%]再买，涨到[昨收+2%]再卖

│   │  判断：偏弱，但未破位
│   │  操作：准备买入，等企稳
│   └─ 开盘后观察5分钟，企稳则在[昨收-2%]挂单买入
│
├─ 情景6：[昨收-4%] < [竞价价] ≤ [昨收-2.5%]

    └─ 如果开盘后5分钟内反弹到[昨收-2%]以上，是假跌
       如果继续下跌到[止损位]以下，准备止损
```

### 第四步：开盘操作（9:30-10:00）

根据开盘价和盘前研判结果，分情景操作：

#### 情景A：[开盘价] > [加权成本]

```
9:30  开盘，不追高
9:31  观察前1分钟走势
      ├─ 继续拉升 → 可能冲高，持有等高点
      └─ 快速回落 → 获利盘出逃，警惕
9:35  二次确认
      ├─ 站稳[加权成本] → 继续持有
      ├─ 跌破[加权成本] → 观望
      └─ 涨到[成本+2%] → 挂卖出[持仓数量×20%]份锁定利润
10:00 方向确认
      ├─ 持续在[加权成本]上方 → 持有，等更高位置
      └─ 跌回[加权成本]下方 → 今日不操作，等明天
```

#### 情景B：[昨收+2%] < [开盘价] ≤ [加权成本]


## 做T操作方案

> **做T买入位和卖出位基于实时价格动态计算，禁止使用固定值。**

### 买入操作
```
条件：[当前价]从当日高点回落2%以上 且 出现企稳信号
挂单：[当前价]×0.98 买入 [做T可用资金上限]÷([当前价]×0.98) 份
金额：[做T可用资金上限]元（= [可用资金]×80%，不满仓做T）
成功后：持仓增加，成本下降
```

> **C8：做T资金设80%上限（代码已实现，见 `[做T可用资金上限]` 变量）。**
> 不将可用资金全部打满用于做T，避免判断错误后没有应急资金应对更大幅度
> 的波动或补仓需要。

### 卖出操作
```
条件：[当前价]从买入价上涨3-4% 且 出现滞涨信号
挂单：买入价×1.03 卖出做T部分
成功后：持仓恢复原数量，成本下降
```

### 做T收益计算（动态）
```
单次做T收益 = (卖出价 - 买入价) × 做T份额
做T后新成本 = [加权成本] - 单次做T收益 ÷ [持仓数量]
```

### 做T纪律

| 规则 | 说明 |
|------|------|
| 买了就挂卖单 | 不要T着T着变成满仓 |
| 每天只做一个方向 | 买了就等卖，不要同天又买又卖 |
| 赚3-4%就走 | 不贪心，积少成多 |
| 设好止损线 | 跌破[止损位]停止做T |
